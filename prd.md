## 3. 디자인 시스템

CLAUDE.md의 Ultimate Dark Mode 팔레트 전체 적용.

```css
:root {
  --bg-primary: #111111;
  --bg-secondary: #1A1A1A;
  --bg-surface: #222222;
  --text-primary: #FFFFFF;
  --text-secondary: #AAAAAA;
  --text-muted: #555555;
  --border: #333333;
}
```

- 폰트: Pretendard (한국어 우선) / Inter (영문 폴백)
- 화려한 컬러, 그라디언트, 과도한 shadow 일체 금지
- 계층은 밝기(opacity/brightness)로만 구분

## 4. 진입 화면 (Entry Screen)

앱 실행 시 가장 먼저 보이는 화면.

### 구성
- 화면 중앙에 단 하나의 질문:
  ```
  오늘, 단 하나만 한다면?
  ```
- 텍스트 입력창 (placeholder: "오늘의 원 띵을 입력하세요")
- CTA 버튼: `집중 시작하기`
- 이미 오늘 원 띵이 저장된 경우 → 대시보드로 바로 진입

### 저장
- `localStorage` key: `oneBlank_today` `{ date, oneThing }`
- 날짜가 바뀌면 자동 초기화

## 5. 핵심 기능 명세

전체 기능은 단일 페이지 스크롤 구조로 배치. 섹션 순서:
1. 안티 비전 고정
2. AI 감정 배출
3. 원 띵 (오늘의 집중 1개)
4. 뽀모도로 타이머
5. 누적 ROI 계산기

---

### 5-1. 안티 비전 (Anti-Vision)

> "지금 이 순간, 흔들리지 않는 하나."

**목적:** 외부 자극(SNS, 타인의 성공 등)에 흔들리지 않고 오늘의 원 띵에 집중시키는 고정 장치.

**동작 방식:**
- 오늘 입력한 원 띵을 화면 최상단에 크고 선명하게 고정 표시
- 날짜별로 자동 초기화 (매일 아침 새로 확인)
- 아침 첫 접속 시 원 띵이 화면 전체를 덮는 풀스크린 모멘트로 표시 (1회)
- 이후 스크롤해도 상단 고정 바(sticky header)에 항상 노출

**UI:**
- 배경: `#1A1A1A`, 텍스트: `#FFFFFF`, 자간 넓게
- "오늘의 원 띵: [내용]" 형식

---

### 5-2. AI 감정 배출 (Emotion Drain)

> "쏟아내. 5초 후엔 사라진다."

**목적:** 감정 쓰레기를 안전하게 비우는 공간. 기록되지 않으므로 솔직하게 쏟아낼 수 있음.

**동작 방식:**
1. 사용자가 감정/생각을 자유롭게 텍스트 입력
2. 입력이 멈추면(또는 엔터 시) Claude API 호출
3. 삭제 전: AI가 1줄 공감 멘트를 실시간으로 화면에 표시
   - 프롬프트 방향: 판단 없이 감정을 수용하는 따뜻한 1문장
4. 5초 카운트다운 (시각적으로 표시)
5. 5초 후 텍스트 완전 삭제 + AI가 원 띵 한 줄 제안
   - 프롬프트 방향: "지금 이 감정을 이겨낼 오늘의 원 띵은 [X]이(가) 아닐까요?"
6. 삭제된 텍스트는 localStorage 포함 어디에도 저장하지 않음

**UI:**
- 텍스트 에어리어: 배경 `#222222`, 테두리 `#333333`
- 카운트다운: 숫자 + 진행 바 (fade-out 애니메이션)
- AI 응답: 이탤릭체, `#AAAAAA` 컬러

---

### 5-3. 원 띵 (One Thing)

> "딱 하나. 그것만."

**목적:** 오늘 완료해야 할 핵심 태스크 1개를 항상 가시화.

**동작 방식:**
- 진입 화면에서 입력한 원 띵이 이 섹션에도 표시
- 완료 버튼: 클릭 시 취소선 + `#555555` 처리 + 완료 시각 기록
- 재설정 버튼: 오늘 원 띵 수정 가능 (하루 1회 제한 또는 자유)

**저장:**
- `localStorage` key: `oneBlank_today`

---

### 5-4. 뽀모도로 타이머 (Pomodoro)

> "25분. 불 꺼. 시작."

**스펙:**
- 집중 시간: 25분 고정
- 휴식 시간: 5분 고정
- 클릭 한 번으로 시작

**동작 방식:**
1. 시작 버튼 클릭 → 25분 카운트다운
2. 25분 완료 → 브라우저 알림 + 소리(optional) + 5분 휴식 자동 시작
3. 휴식 완료 → 다시 25분 준비 상태
4. 세션 완료 시 완료 횟수 +1, localStorage에 누적 저장
5. 오늘 완료 세션 수 표시 (예: "오늘 3세션 완료 · 총 75분")

**저장:**
- `localStorage` key: `oneBlank_pomodoro` `{ date, sessions, totalMinutes }`

**UI:**
- 타이머 숫자: 대형 폰트, 화면 중앙 배치
- 상태 표시: 집중 중 / 휴식 중 / 준비
- 진행 링 또는 진행 바 (subtle, 화이트 계열)

---

### 5-5. 누적 ROI 계산기

> "내 시간의 가치를 숫자로."

**목적:** 뽀모도로로 집중한 시간이 실제 수익으로 얼마인지 가시화. 동기부여.

**입력값:**
- 시간당 단가 (원, 사용자 입력, localStorage 저장)

**계산식:**
```
오늘 수익 = (오늘 완료 뽀모도로 세션 × 25분 / 60) × 시간당 단가
누적 수익 = 전체 완료 세션 합산으로 동일 계산
```

**표시:**
- 오늘 집중 시간 / 오늘 수익 / 누적 집중 시간 / 누적 수익
- 단가 미입력 시: "시간당 단가를 설정하면 수익을 계산해 드려요" 안내

**저장:**
- `localStorage` key: `oneBlank_roi` `{ hourlyRate }`
- 수익은 pomodoro 세션 데이터 기반 실시간 계산 (별도 저장 불필요)

## 7. Claude API 연동 명세

| 용도 | 모델 | 프롬프트 방향 |
|---|---|---|
| 감정 공감 멘트 | claude-sonnet-4-6 | "판단 없이 감정을 수용하는 따뜻한 한국어 1문장. 50자 이내." |
| 원 띵 제안 | claude-sonnet-4-6 | "사용자가 방금 쏟아낸 감정을 바탕으로, 오늘 집중할 원 띵을 한 줄로 제안. 20자 이내." |

- API 키: 환경변수 `ANTHROPIC_API_KEY` (Vercel 환경변수에 등록)
- 호출 위치: Next.js API Route `/api/emotion`
- 감정 텍스트는 API 호출 후 응답 수신 즉시 서버에서도 폐기 (저장 없음)
